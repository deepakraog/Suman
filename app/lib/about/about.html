AngularJS 1.6 with ES6

Why ES6?

AngularJS version 1.6 with ES5 is the most stable version for production among all. Around 2 years back Angular was the daddy of all JS frameworks. It has to see a downfall because of the introduction of ES6. ES6 is nothing but the standard of writing javascript. What we are using currently is ES5 on which most of us are familiar with. There is an assosiation in Europe called ECMA (European Computer Manufacturers Association) wanted to intoduce oops concepts in Javascript and the result is ES6. Currently our browsers are not equipped to understand and run the code written using ES6 syntax so we need a transpiler to convert ES6 code to corresponding ES5 code and feed it to browsers. Babel is the famous one among the current transpilers. In future these transpilers will go off as our browers get equipped to run the ES6 code. Browsers will continue to evolve and adopt features in parts. Best to start supporting things now. S0 angular team decided to support coders to write code with ES6 syntax and this is how it evolved.

SPA through routing

AngularJS allows you to create Single Page Applications which in turn greatly increases the performance of your application. Let's see what is SPA, In our normal/conventional application if you want to switch from one view to other the whole page has to be re-rendered and that call to the server is very heavy which put load on server which results the slow response. If you make a request what you will get as a response is a whole new HTML/JSP page which includes images, corresponding css and js ﬁles which is a heavy process. On the other hand, In Single Page Applications if you make a request to switch between the views what you will get as a response is just a template in the form of JSON data and it is very lite process exactly like AJAX calls. This can be achieved through Routing. Routing is noting but configuration.

2-way data binding

AngularJS is famous among the JS framewoks because of it's 2-way data binding feature. It made very ﬂexible to work with large data passing data from fontend to backend. There is a concept called $scope which acts as a glue for application data. Using $scope whatever the data you change in the frontend will reﬂect in the JS ﬁle and Viceversa.

Directives

Directive is one of the powerfull tool of AngularJS. If you have a repeated code in your application you can put that code in a directive and call that within a single line of code. Ex: if I have 7-8 tables in my application, Rather than writing table,thead,tbody,tr,th,td tags everytime what I can do is make that table as a directive and use that directive within a single line where ever it is necessary by passing the data from the controller. This concept is called Component in Angular2 and React18.

SASS

Intro

SASS (Syntactically Awesome Stylesheet) is a CSS pre-processor, which helps to reduce repetition with CSS and saves time. It is more stable and powerful CSS extension language that describes the style of document structurally. This tutorial covers the basics of SASS.

Advantages

  • It allows writing clean CSS in a programming construct. It helps in writing CSS quickly.
  • It is a superset of CSS, which helps designers and developers work more efficiently and quickly.
  • As Sass is compatible with all versions of CSS, we can use any available CSS libraries.
  • It is possible to use nested syntax and useful functions such as color manipulation, mathematics and other values.

Features of SASS

It is more stable, powerful, and compatible with versions of CSS. It is a super set of CSS and is based on JavaScript. It is known as syntactic sugar for CSS, which means it makes easier way for user to read or express the things more clearly. It uses its own syntax and compiles to readable CSS. You can easily write CSS in less code within less time. It is an open source pre-processor, which is interpreted into CSS.
